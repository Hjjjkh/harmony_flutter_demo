# 华为官方文档和约束规则

## 📚 目录说明

此目录用于存放华为官方文档和开发约束规则，帮助 AI 更好地理解华为鸿蒙开发规范。

## 📁 目录结构

```
docs/HUAWEI_DOCS/
├── README.md                    # 本文档
├── official/                    # 官方文档
│   ├── harmonyos/              # OpenHarmony 官方文档
│   ├── deveco/                 # DevEco Studio 文档
│   └── flutter/                # Flutter for OpenHarmony 文档（如果有）
├── constraints/                 # 约束规则
│   ├── coding_standards.md     # 编码规范
│   ├── api_guidelines.md       # API 使用指南
│   ├── design_patterns.md      # 设计模式
│   └── best_practices.md       # 最佳实践
└── examples/                    # 示例代码
    ├── platform_channel/       # Platform Channel 示例
    └── native_features/        # 原生功能示例
```

## 📖 如何添加文档

### 方式 1: 直接添加文件

1. 将文档文件复制到相应目录
2. 更新本文档的索引
3. 提交到 Git

### 方式 2: 通过 AI 添加

在对话框中描述：
```
添加华为官方文档：[文档内容或链接]
```

AI 会：
- 创建相应的文档文件
- 更新索引
- 组织文档结构

### 方式 3: 添加链接

如果文档是外部链接，可以创建引用文件：
```markdown
# 文档标题
链接: https://developer.harmonyos.com/...
描述: ...
```

## 🎯 使用方式

### 在开发中引用

AI 在编写代码时会：
1. 参考官方文档
2. 遵循约束规则
3. 应用最佳实践
4. 参考示例代码

### 更新文档

当有新的官方文档或规则时：
1. 添加到相应目录
2. 更新索引
3. 通知团队

## 📝 文档格式

- 使用 Markdown 格式

- 包含代码示例

### 代码示例
